# Terroir — Implementation Roadmap

## Architecture

### System Overview

```
[Data Pipeline — Python 3.11, runs once offline, not part of iOS build]
  FAO HWSD v2.0 (soil taxonomy)     ──┐
  WorldClim 2.1 BIO (climate)        ├─→ 02_rasterize.py → 0.5° WGS84 grid (360×720)
  MODIS MCD12Q1 (land cover/NDVI)    │                         ↓
  GMTED2010 (elevation)              │               03_flavor_engine.py
  GLiM (geology/water mineralogy)   ──┘          (weight matrix × 5 sources → 12D vector)
                                                              ↓
                                                   04_validate.py (20 canonical locations)
                                                              ↓
                                                   05_encode_binary.py
                                                              ↓
                                               terroir.bin (FlatBuffers + LZ4, ~45MB)
                                                              ↓
                                               bundled → Terroir/Resources/terroir.bin

[iOS App — Swift/SwiftUI + SceneKit]
  GlobeView (SceneKit)
       │ tap gesture → SCNHitTest → hit normal → GeoCoordinate
       ↓
  FlavorLookup.swift
       │ gridCellIndex = ((lat+90)/0.5)*720 + ((lon+180)/0.5)
       │ memory-mapped terroir.bin → FlatBuffers decode → FlavorCell
       ↓
  FlavorVector (12 floats) → dominantNotes (top notes > 0.35 threshold)
       ↓
  TemplateEngine.swift → FlavorDescription (prose string)
       ↓
  FlavorCardView (SwiftUI overlay)
       │ RadarChartView (Canvas, 12-axis)
       │ CLGeocoder (async, non-blocking location name)
       │ "More Detail" button ──→ EnrichmentService.swift ──→ CloudKit/Vercel API
       ↓
  ShareCardView (ImageRenderer → 1080×1080 PNG → ShareLink) [Phase 3]
```

### Data Model

**FlatBuffers Schema (`data-pipeline/terroir.fbs`):**
```fbs
namespace Terroir;

table FlavorCell {
  // 12 flavor dimensions, normalized 0.0–1.0
  earthy:      float;
  mineral:     float;
  bright:      float;
  citric:      float;
  floral:      float;
  herbaceous:  float;
  smoky:       float;
  woody:       float;
  saline:      float;
  tannic:      float;
  vegetal:     float;
  aromatic:    float;
  // Source metadata (used for More Detail context)
  soil_class:     ubyte;   // 0–28 FAO soil classes mapped to 6 archetypes
  climate_zone:   ubyte;   // 0–30 Köppen codes mapped to 8 bands
  ndvi_band:      ubyte;   // 0=barren, 1=sparse, 2=grassland, 3=shrubland, 4=dense forest
  elevation_band: ubyte;   // 0=sea level, 1=0–500m, 2=500–1500m, 3=1500–3000m, 4=3000m+, 5=4000m+
  is_ocean:       bool;
  ocean_zone:     ubyte;   // 0=polar (|lat|>60), 1=temperate (30–60), 2=tropical (<30) — valid if is_ocean
}

table FlavorGrid {
  cells:     [FlavorCell];  // Length = 259,200 (360 lat bands × 720 lon bands)
  lat_cells: uint;          // 360
  lon_cells: uint;          // 720
  version:   uint;          // Schema version — increment on breaking changes
}

root_type FlavorGrid;
```

**Swift Type Definitions:**

```swift
// Models/GeoCoordinate.swift
struct GeoCoordinate {
    let latitude: Double    // -90.0 to +90.0
    let longitude: Double   // -180.0 to +180.0

    // 0.5° resolution: 360 lat bands × 720 lon bands = 259,200 cells
    var gridCellIndex: Int {
        let latBand = Int(min(max((latitude + 90.0) / 0.5, 0), 359))
        let lonBand = Int(min(max((longitude + 180.0) / 0.5, 0), 719))
        return latBand * 720 + lonBand
    }
}

// Models/FlavorDimension.swift
enum FlavorDimension: String, CaseIterable {
    case earthy, mineral, bright, citric, floral, herbaceous
    case smoky, woody, saline, tannic, vegetal, aromatic

    var displayLabel: String { rawValue.capitalized }
    // Axis angle for radar chart: 30° per dimension
    var radarAngle: Double { Double(Self.allCases.firstIndex(of: self)!) * (360.0 / 12.0) }
}

// Engine/FlavorVector.swift
struct FlavorVector {
    let earthy, mineral, bright, citric: Float
    let floral, herbaceous, smoky, woody: Float
    let saline, tannic, vegetal, aromatic: Float

    var asArray: [(FlavorDimension, Float)] {
        zip(FlavorDimension.allCases,
            [earthy, mineral, bright, citric, floral, herbaceous,
             smoky, woody, saline, tannic, vegetal, aromatic]).map { ($0, $1) }
    }

    // Top notes with value > 0.35 threshold, sorted descending
    var dominantNotes: [FlavorDimension] {
        asArray.filter { $0.1 > 0.35 }
               .sorted { $0.1 > $1.1 }
               .prefix(5)
               .map { $0.0 }
    }

    // Mock for Xcode Previews and unit tests
    static let mockBurgundy = FlavorVector(
        earthy: 0.82, mineral: 0.74, bright: 0.31, citric: 0.20,
        floral: 0.18, herbaceous: 0.25, smoky: 0.15, woody: 0.30,
        saline: 0.08, tannic: 0.68, vegetal: 0.22, aromatic: 0.28
    )
}

// Models/EnrichmentData.swift
struct EnrichmentData: Codable {
    let dominantCrop: String?           // "Pinot Noir grape" / nil if unknown
    let waterSource: String?            // "limestone aquifer" / "glacial melt"
    let waterMineralCharacter: String?  // "hard, calcite-rich" / "soft, neutral"
    let pairingRegion: String?          // "similar to Burgundy, France"
    let pairingSuggestion: String?      // "aged Comté, mushroom risotto, duck confit"
    let localNote: String?              // One surprising local geographic fact
}

// AppState.swift
@Observable
class AppState {
    var selectedCoordinate: GeoCoordinate? = nil
    var flavorVector: FlavorVector? = nil
    var locationName: String = "Loading..."
    var isLoadingEnrichment: Bool = false
    var enrichmentData: EnrichmentData? = nil
    var activeOverlay: GlobeOverlay = .none
    var showCard: Bool = false
}

enum GlobeOverlay: String, CaseIterable {
    case none = "None"
    case soil = "Soil"
    case climate = "Climate"
    case vegetation = "Vegetation"
}
```

### API Contracts

**Enrichment API:**

| Field | Value |
|-------|-------|
| Primary endpoint | CloudKit Function (prototype first — zero infra cost) |
| Fallback endpoint | `https://terroir-enrich.vercel.app/api/enrich` |
| Method | GET |
| Auth | None in v1 — Vercel IP rate limiting (100 req/day/IP on hobby tier) |
| Parameters | `?lat={float}&lon={float}` |
| Response type | `EnrichmentData` JSON |
| Timeout | 5 seconds — silent failure, base card stays visible |
| Cache | In-memory, keyed by `gridCellIndex` — same location never re-fetches in session |
| Error handling | Any non-200 or timeout → `enrichmentData = nil` → hide "More Detail" button silently |

**Response Shape:**
```json
{
  "dominantCrop": "Pinot Noir grape",

  "pairingSuggestion": "aged Comté, mushroom risotto, duck confit",
```

## Scope Boundaries

**In scope (v1):**
- 3D SceneKit globe with pan/rotate gesture and tap-to-coordinate
- Offline flavor card: radial chart + templated prose from bundled `terroir.bin`
- 12-dimension flavor vector per cell: earthy, mineral, bright, citric, floral, herbaceous, smoky, woody, saline, tannic, vegetal, aromatic
- Ocean handling (3 maritime bands)
- "Use My Location" via CoreLocation
- "More Detail" enrichment via CloudKit/Vercel API (async, non-blocking)
- Data overlay toggle: Soil / Climate / Vegetation (pre-baked PNG textures)
- Share card: 1080×1080 PNG via ImageRenderer + ShareLink
- App Store submission at $4.99 one-time

**Out of scope (v1 — do not implement):**
- Comparative tasting (two locations side by side)
- Terroir walk (route-tracing flavor journey)
- Wine/product pairing database integration
- User accounts, saved locations, or any user-generated data
- Seasonal variation (v1 uses annual averages)
- AR camera mode
- Subscription or IAP
- Android / web version

**Deferred to v2:**
- Comparative tasting view
- Seasonal flavor modes (spring/summer/autumn/winter)
- Crowdsourced accuracy ratings
- Wine region pairing database links
- Expanded enrichment dataset (v1 covers ~500 curated locations; v2 aims for global coverage via automated crop/geology inference)

## Security & Credentials

- No user accounts, no authentication in v1
- Location data: CoreLocation `.whenInUse` only. Coordinate used on-device to compute grid cell index. Location never transmitted to any server.
- Enrichment API: only `lat` and `lon` floats sent — no device ID, no user ID, no IP logged beyond Vercel's default edge logs (acceptable at this scale)
- No API keys bundled in the app; enrichment endpoint is unauthenticated in v1
- Privacy nutrition label: Location (on-device only, not linked to user, not used for tracking)
- No third-party SDKs that require privacy disclosure (no Firebase, Amplitude, Crashlytics)
- Use Apple MetricKit + App Store Connect for crash reporting (zero configuration required)

## Phase 0: Data Pipeline (Weeks 1–2)

**Objective:** Python pipeline produces `terroir.bin` (FlatBuffers + LZ4, < 60MB) that passes validation against all 20 canonical locations. No Xcode involved. `terroir.bin` is the only deliverable.

**Tasks:**

1. Set up Python environment and install all dependencies.
   **Acceptance:** `python -c "import rasterio, flatbuffers, lz4; print('OK')"` prints `OK` with no import errors.

2. Download all source data via `01_download_sources.sh`:
   - FAO HWSD v2.0 soil raster → https://www.fao.org/soils-portal/soil-survey/soil-maps-and-databases/harmonized-world-soil-database-v20/en/ (free, registration required)
   - WorldClim 2.1 BIO variables at 2.5 arc-minute → https://worldclim.org/data/worldclim21.html (direct download, ~350MB)
   - MODIS MCD12Q1 annual land cover → NASA Earthdata (registration at https://urs.earthdata.nasa.gov/ required)
   - GMTED2010 elevation at 30 arc-second → https://topotools.cr.usgs.gov/gmted_viewer/ (USGS, direct download)
   - GLiM global lithological map → https://www.geo.uni-hamburg.de/geologie/forschung/geochemie/glim.html
   **Acceptance:** All 5 source raster files present in `data-pipeline/raw/`, each > 1MB.

3. Run `02_rasterize.py`: resample all 5 sources to a common 0.5° WGS84 grid (360 lat × 720 lon = 259,200 cells). Output: `data-pipeline/intermediate/grid_{source}.npy` (numpy float32 arrays).
   **Acceptance:** `python 02_rasterize.py --validate` prints `All grids shape-matched: (360, 720) ✓`. No NaN values in any grid.

4. Run `03_flavor_engine.py`: apply weight matrix (SOIL + CLIMATE + ELEVATION + NDVI) per cell. Add coastal saline bonus (cells within 100km of coastline). Set ocean profiles for ocean cells. Output: `data-pipeline/intermediate/flavor_grid.npy` shape `(259200, 12)`, dtype float32, all values in [0.0, 1.0].
   **Acceptance:** `python -c "import numpy as np; g=np.load('intermediate/flavor_grid.npy'); print(g.shape, g.min(), g.max(), np.isnan(g).sum())"` prints `(259200, 12) 0.0 1.0 0`.

5. Run `04_validate.py`: load `validation_set.json`, look up each of the 20 canonical locations, compare actual `dominantNotes` vs `expected_top3`. Print pass/fail per location.
   **Acceptance:** `python 04_validate.py` outputs ≥ 17/20 `PASS`. Tune weight matrix in `03_flavor_engine.py` until this passes. Document which 3 locations fail and why (acceptable misses should be geologically ambiguous areas).

6. Run `05_encode_binary.py`: serialize `flavor_grid.npy` into FlatBuffers format per `terroir.fbs` schema, apply LZ4 compression (lz4_frame format, not lz4_block — iOS's `COMPRESSION_LZ4` expects lz4_frame). Output: `terroir.bin`.
   **Acceptance:** `ls -lh terroir.bin` shows size between 20MB and 60MB. `python 05_encode_binary.py --verify` reads back cell at gridCellIndex 0 and prints non-zero float values for at least 3 dimensions.

7. Run `06_bake_overlays.py`: render soil_class, climate_zone, and ndvi_band grids to color-mapped 2048×1024 PNG images using perceptually distinct palettes (colorbrewer-safe). Output: `terroir-ios/Terroir/Resources/overlay_soil.png`, `overlay_climate.png`, `overlay_vegetation.png`.
   **Acceptance:** Each PNG opens in Preview, shows visible regional variation, is exactly 2048×1024 pixels.

8. Copy `terroir.bin` to `terroir-ios/Terroir/Resources/terroir.bin`.
   **Acceptance:** File exists at that path, size matches the one from step 6.

**Verification Checklist:**
- [ ] `python -c "import rasterio, flatbuffers, lz4; print('OK')"` → `OK`
- [ ] `python 02_rasterize.py --validate` → `All grids shape-matched: (360, 720) ✓`
- [ ] `python -c "import numpy as np; g=np.load('intermediate/flavor_grid.npy'); print(g.shape, g.min(), g.max(), np.isnan(g).sum())"` → `(259200, 12) 0.0 1.0 0`
- [ ] `python 04_validate.py` → ≥ 17/20 PASS
- [ ] `ls -lh terroir.bin` → 20MB–60MB
